## 项目概述

本项目是一个完整的**六角星跳棋（Chinese Checkers）**桌面游戏，由 5 个 Python 模块构成，涵盖棋盘逻辑、回合状态机、工业级 AI 搜索与 Tkinter 图形界面。

---

## 模块结构

```
chinese_checkers/
├── main.py    # 启动入口：创建窗口，进入事件循环
├── board.py   # 棋盘数据结构与移动规则（坐标系、连跳搜索）
├── game.py    # 游戏控制器（三状态状态机、悔棋、胜负判定）
├── ai.py      # AI 对手（迭代加深 Alpha-Beta + 多维评估函数）
└── gui.py     # Tkinter 图形界面（绘制、坐标转换、交互事件）
```

### `board.py` — 棋盘数据结构与移动规则

#### 坐标系与六角网格映射

棋盘使用行列整数坐标（行号 0–16）。六角星的物理交错结构通过**相邻行列号奇偶性互换**来模拟——偶数行只有偶数列，奇数行只有奇数列：

```
行 0:          [12]           ← N 角顶点
行 1:        [11, 13]
行 2:      [10, 12, 14]
行 3:    [ 9, 11, 13, 15]
行 4:  [ 0,  2,  4, ...24]   ← 全偶数列（最宽行）
行 5:  [ 1,  3,  5, ...23]   ← 全奇数列
...
行 8:  [ 4,  6,  8, ...20]   ← 主板中间行
...
行 16:         [12]           ← S 角顶点
```

六个方向的偏移量为 `DIRECTIONS = [(-1,-1),(-1,1),(0,-2),(0,2),(1,-1),(1,1)]`：
- **列偏移 ±2**（水平相邻）：同行内移动，列号奇偶性保持不变
- **列偏移 ±1，行偏移 ±1**（对角相邻）：跨行时列号奇偶随行号翻转，±1 正好维持这一特性

#### 三角区域定义

六个角三角区（N/S/NE/NW/SE/SW）通过 `_TRIANGLE_BOUNDS` 硬编码**每行的列范围边界**来界定：

| 三角区 | 行范围 | 示例（行→列范围） |
|---|---|---|
| N | 0–3 | `{0:(12,12), 1:(11,13), 2:(10,14), 3:(9,15)}` |
| S | 13–16 | `{13:(9,15), 14:(10,14), 15:(11,13), 16:(12,12)}` |
| NW | 4–7 | `{4:(0,6), 5:(1,5), 6:(2,4), 7:(3,3)}` |
| NE | 4–7 | `{4:(18,24), 5:(19,23), 6:(20,22), 7:(21,21)}` |
| SW | 9–12 | `{9:(3,3), 10:(2,4), 11:(1,5), 12:(0,6)}` |
| SE | 9–12 | `{9:(21,21), 10:(20,22), 11:(19,23), 12:(18,24)}` |

初始化时遍历这些边界，将每个三角区的合法格子收集为 `set` 存入 `self.triangles`，后续区域判断均为 O(1) 集合查询。

#### 移动规则

| 规则 | 方法 | 说明 |
|---|---|---|
| 单步移动 | `get_simple_moves` | 遍历六方向，目标格在棋盘内且为空则合法 |
| 单次跳跃 | `get_single_jumps` | 中间格有棋子、落点格为空，跨越距离为移动的 2 倍 |
| 递归连跳 | `_find_jumps` / `get_all_jump_destinations` | DFS 递归展开所有可达跳跃目标，`visited` 集合防止循环 |
| 目标区限制 | `get_all_valid_moves` | 棋子已进入目标三角区时，过滤掉所有目标区外的目标格 |

连跳搜索时，`get_all_jump_destinations` 会先将棋子临时从棋盘移走再递归，避免棋子被自身阻挡。

### `game.py` — 游戏控制器

#### Player 类

封装单名玩家的所有属性：

| 属性 | 说明 |
|---|---|
| `id` | 玩家编号（1 或 2），与 `board.grid` 中存储的值对应 |
| `name` | 显示名称（"玩家 1" / "玩家 2" / "电脑"） |
| `home` | 起始三角区名称（默认：玩家 1 → `'S'`，玩家 2 → `'N'`） |
| `goal` | 目标三角区名称（与 `home` 相对） |
| `color` | 界面显示颜色（十六进制字符串） |
| `is_ai` | 是否由 AI 控制 |

#### 回合状态机

`Game` 维护三状态 FSM，所有交互通过唯一入口 `select(pos)` 分派：

```
                    ┌──────────────────────────────────────┐
                    ▼                                      │
     ┌──────────────────────────────┐                     │
     │            IDLE              │                     │
     │  等待玩家选择己方棋子          │                     │
     └──────────────────────────────┘                     │
                    │ 点击己方棋子                          │
                    ▼                                      │
     ┌──────────────────────────────┐                     │
     │       PIECE_SELECTED         │                     │
     │  展示合法目标格               │──── 点击其他处 ──────┤
     └──────────────────────────────┘                     │
                    │ 点击目标格                            │
                    ▼                                      │
             _execute_move()                               │
                    │                                      │
          ┌─────────┴─────────┐                           │
          │ 是跳跃且有后续跳？  │                           │
          └─────────┬─────────┘                           │
         yes        │         no                          │
          │                    │                          │
          ▼                    ▼                          │
┌──────────────────┐    _finish_turn()                    │
│  CHAIN_JUMPING   │    回合结束，切换玩家 ──────────────────┘
│  继续跳或结束跳跃  │
└──────────────────┘
          │ end_jump()
          └──────────► _finish_turn()
```

#### 连跳合并机制

连跳过程中 `move_history` 始终只维护**一条**记录，每次新跳将记录的终点更新为最新落点（`orig_from → to_pos`），确保整段连跳作为一次操作整体悔棋。`jump_visited` 集合记录本回合已访问的位置，防止连跳路径回头。

#### 胜负判定

`_check_win` 检查目标三角区的所有格子是否全部被当前玩家棋子占满。每次 `_finish_turn` 后触发，满足条件则设置 `game_over = True` 并记录 `winner`。

#### 悔棋

`undo` 从 `move_history` 弹出最后一条 `(from, to, player_idx)` 记录，反向移动棋子并回退回合指针。连跳进行中禁止悔棋，避免历史记录处于不完整状态。

#### 重置与模式切换

`reset(ai_mode)` 清空棋盘、重新摆子、重置所有状态字段，并根据 `ai_mode` 决定玩家 2 的名称和 `is_ai` 属性。GUI 的模式单选按钮每次切换都直接调用 `reset`，立即生效。

### `ai.py` — 搜索算法

#### 迭代加深与 Alpha-Beta 剪枝

`choose_move` 以迭代加深方式运行，深度从 1 逐层推进至上限 13，每轮受 **2.8 秒**硬超时约束。超时时立即放弃当前深度，使用上一层的最优结果。

```
choose_move()
    │
    ├── 迭代加深（深度 1 → 13，2.8 秒超时）
    │       │
    │       ├── 走法按 _quick_score() 启发排序
    │       │     行进度×15 + 跳跃距离×3 + 进入目标区+100 + 移动掉队棋子+40
    │       │
    │       └── _alphabeta(depth, α, β)
    │               ├── 查置换表（EXACT / LOWERBOUND / UPPERBOUND）
    │               ├── depth=0 → _evaluate()
    │               ├── Alpha-Beta 剪枝（α ≥ β 时截断）
    │               └── 写回置换表
    │
    └── 对每个候选走法：减去 _oscillation_penalty()
```

| 搜索技术 | 说明 |
|---|---|
| 迭代加深 | 深度 1 开始逐层加深，时间预算内尽量搜更深 |
| Alpha-Beta 剪枝 | α ≥ β 时剪枝，大幅减少节点展开数 |
| 置换表 | 以 `hash(p1_positions, p2_positions, is_maximizing)` 为键缓存，含 EXACT / LOWERBOUND / UPPERBOUND 三种标志 |
| 走法排序 | 每层展开前按启发分排序，好走法靠前最大化剪枝效率 |
| 反振荡惩罚 | 维护长度 8 的历史局面队列，走法导致局面与近期重复则扣分（越近惩罚越重） |
| 时间控制 | 每步限 2.8 秒，`_search_aborted` 标志传递中止信号 |
| 内存管理 | 置换表超 30 万条时整体清空，防止内存无限增长 |

等分走法之间随机选择，避免 AI 行为可预测。若搜索到得分超过 9500 的必胜走法，提前终止迭代。

### `ai.py` — 评估函数

`_evaluate()` 返回 AI 方得分减对手得分。单方评分由 `_score_player()` 计算，综合以下八个维度：

| 维度 | 权重 | 说明 |
|---|---|---|
| 行进度 | `progress × 12.0` | 最核心项，棋子行号越靠近目标方向得分越高 |
| 中心性 | `-abs(col - 12) × 1.2` | 中场阶段惩罚偏离中心列（第 12 列为水平中心），引导棋子走跳跃路径最多的走廊 |
| 目标区占领 | `80 + depth × 15` | 已进入目标三角区的棋子按深度额外加分，最深处可得 80 + 3×15 = 125 |
| 最近空目标格距离 | `-hex_dist × 5.0` | 对目标区外的棋子，惩罚其到最近空目标格的六角距离，驱动填满目标区 |
| 终局紧迫度 | `-hex_dist × (1 + e^(n-6))` | 目标区内棋子数 ≥ 6 后，掉队棋子的距离惩罚指数级增长 |
| 掉队惩罚 | `-(16 - min_progress) × (10 + n×4)` | 最落后棋子惩罚权重随已入目标数线性增大，终局阶段极为严厉 |
| 散布惩罚 | `-spread^1.4 × 5.0` | 最大行进度与最小行进度之差的 1.4 次幂，惩罚队形过于分散 |
| 桥接奖励 | `相邻棋子对数 × 4.0` | 统计所有相邻棋子对，奖励紧凑阵型（有利于构建连跳链） |

六角距离由 `_hex_dist` 计算，处理了行列两套轴的投影关系：同向时取绝对值之和，异向时取最大绝对值。

### `gui.py` — Tkinter 图形界面

#### 棋盘绘制

每次操作后 `_draw_board()` 先清空 Canvas 再全量重绘，绘制顺序为：

1. **三角区底色**（`_draw_triangle_zones`）：对每个三角区的每个格子，绘制带 `stipple='gray25'` 参数的半透明圆形色斑，形成区域着色但不遮挡格子。
2. **网格连线**：遍历所有格子及其六向邻居，用 `(min(pos, nb), max(pos, nb))` 元组去重，每条边只画一次，避免重叠线段。
3. **棋子与空位**：有棋子调用 `_draw_piece`（三层叠绘），空位绘制小圆点（`HOLE_R = 5`）。
4. **选中高亮**：已选棋子外围绘制黄色（`#f1c40f`）虚线圆圈（`dash=(4,2)`）。
5. **合法目标标记**：每个合法目标格绘制绿色（`#2ecc71`）虚线圆圈（`dash=(3,3)`）加中心绿点，双重提示可落子位置。

棋子三层叠绘逻辑（`_draw_piece`）：先在右下偏移 1px 绘制深色阴影椭圆，再绘制主色椭圆加深色描边，最后在左上绘制浅色小椭圆作为高光，形成立体感。

#### 坐标转换

| 方法 | 方向 | 实现要点 |
|---|---|---|
| `_pos_to_pixel(r, c)` | 行列 → 像素 | `x = MARGIN_X + c × CELL_SIZE`，`y = MARGIN_Y + r × CELL_SIZE × √3`（六角网格行高比） |
| `_pixel_to_pos(px, py)` | 像素 → 行列 | 遍历所有合法格子，返回欧氏距离最近且距离在 `PIECE_R + 6` 以内的格子；超出范围返回 `None` |

`_pixel_to_pos` 采用**最近距离匹配**而非公式逆变换，对六角网格的不规则布局天然鲁棒，点击容差固定为棋子半径加 6 像素。

#### 右侧控制面板

面板宽度固定 220px，从上至下依次排列：

- **标题区**：「跳 棋」大字 + 「Chinese Checkers」副标题，分隔线分组
- **当前玩家指示器**：彩色圆点（Canvas 绘制，颜色与棋子一致）+ 玩家名称标签
- **状态文本**：`game.status_text` 属性动态提供文案（IDLE / PIECE_SELECTED / CHAIN_JUMPING / 游戏结束四种措辞）
- **功能按钮**：「结束跳跃」「悔 棋」「重新开始」三个按钮，各自在不可用时变灰（`cursor='arrow'`，拒绝鼠标悬停与点击）
- **游戏模式单选**：「双人对战」/ 「人机对战」，切换后立即调用 `reset` 重开
- **退出按钮**：红色，调用 `root.quit`

#### 交互事件

| 事件 | 处理逻辑 |
|---|---|
| 鼠标点击棋盘 | `_on_click` → `_pixel_to_pos` → `game.select(pos)` → `_draw_board()`；若轮到 AI 则 400ms 后触发 `_ai_turn` |
| 「结束跳跃」按钮 | `game.end_jump()` → `_draw_board()`；同样检测是否轮到 AI |
| 「悔 棋」按钮 | `game.undo()` → `_draw_board()` |
| 「重新开始」按钮 | 读取模式单选 → `game.reset(ai_mode)` → `_draw_board()` |
| 按钮悬停 / 点击 | `_make_btn` 用 `Frame + Label` 组合模拟按钮，通过 `<Enter>/<Leave>/<Button-1>` 事件绑定实现颜色切换与短暂灰色闪烁反馈；禁用状态通过 `cursor` 属性（`'hand2'` vs `'arrow'`）判断，不响应悬停和点击 |

AI 回合（`_ai_turn`）在主线程同步运行，`root.after` 仅用于制造视觉延迟，不涉及多线程。AI 选出走法后先 `select(from_pos)` 触发高亮，300ms 后再 `select(to_pos)` 落子；若落点仍处于连跳状态，立即调用 `end_jump()` 结束回合。

#### 与 Game 对象的交互

GUI 不存储任何游戏状态，所有逻辑通过 `self.game` 代理——棋盘格状态、当前玩家、合法目标、游戏是否结束均直接读取 `game` 的属性。每次用户操作或 AI 落子后，统一调用 `_draw_board()` 全量重绘，`_update_status()` 同步刷新面板文字与按钮可用状态。

## 核心特性

- **🎮 双模式** — 双人对战与人机对战，游戏中随时切换，立即重置
- **♟ 连跳状态机** — 三状态 FSM 精确管理连跳流程，连跳中可随时手动结束
- **↩ 整体悔棋** — 连跳全程合并为一条历史记录，一次撤销整段连跳；连跳中禁止悔棋
- **🚧 目标区保护** — 棋子进入目标三角区后不允许跳出，符合标准规则
- **🤖 工业级 AI** — 迭代加深 + Alpha-Beta + 置换表 + 反振荡惩罚，思考时间上限 2.8 秒
- **🎨 三层棋子渲染** — 阴影 + 主色 + 高光叠绘，stipple 半透明三角区底色
- **🖱 最近距离点击识别** — 像素转格子用最近距离匹配，六角不规则布局下无误触

## 模块依赖关系

```
main.py
  └── gui.py  (ChineseCheckersGUI)
        ├── game.py  (Game)          ← 唯一业务逻辑层，持有 Board
        │     └── board.py  (Board)  ← 纯数据层，无上层依赖
        └── ai.py  (SmartAI)         ← 持有 Game 引用，搜索时直接读写 board.grid
```

**设计原则**：`Board` 是无状态数据层；`Game` 是唯一状态中心；`GUI` 零状态，每次操作后全量重绘；`AI` 搜索时临时修改棋盘并还原，搜索结束后棋盘状态不变。
